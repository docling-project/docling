# 가드레일 정규식 필터 프리셋 (#315)

[민감정보 분류/마스킹 가이드](guardrail_workflow_setup.md) 4절 1단계에서 가드레일 인스턴스에
등록하는 **정규식 필터 7종의 복붙용 원본**입니다. 정규식 필터는 GenOS 가드레일 인스턴스가
단일 소스이며, 이 문서는 새 사이트/인스턴스 구성 시 붙여넣을 프리셋과 그 근거를 기록합니다.

- 각 필터: 차단 방식=**정규식**, 응답 방식=**"마스킹 처리하여 제공"**, 치환 규칙=아래 "치환 토큰"
- GenOS 정규식 필터는 원문(구분자 포함)에 대해 Python `re` 로 동작 → 붙은형(01012345678)과
  구분형(010-1234-5678, 공백/점 등)을 모두 커버하도록 작성됨

## UI 입력 방법 (필터 추가 다이얼로그)

1. **제목**: 항목명(예: 주민등록번호)
2. **차단 방식**: **정규식** 선택
3. **정규식** 칸: 아래 원문 블록의 정규식 (한 줄, 줄바꿈 금지)
4. **응답 방식**: **"마스킹 처리하여 제공"** 선택
   - ⚠️ **"사용자에게 안내 메시지 제공"은 차단(BLOCK)** — 걸리면 문서 전체가 안내문구로 통째
     교체됩니다(마스킹 아님). 반드시 "마스킹 처리하여 제공"을 선택할 것.
5. **치환 규칙** 칸: 아래 "치환 토큰"(예: `[주민등록번호]`) 입력
   - 정규식이 매칭한 부분이 이 문자열로 치환됩니다. `[...]` 형태를 유지해야 워크플로우가
     소분류(`specific_category`)까지 복원합니다.

## 정규식 7종 (복붙용)

| 항목(제목) | 치환 토큰 | 정규식 (Python `re`) |
|---|---|---|
| 주민등록번호 | `[주민등록번호]` | `(?<!\d)\d{2}(?:0[1-9]\|1[0-2])(?:0[1-9]\|[12]\d\|3[01])[-./_:() ]*[1-8]\d{6}(?!\d)` |
| 휴대전화번호 | `[휴대전화번호]` | `(?<!\d)(?:010\|011\|016\|017\|018\|019)[-./_:() ]*\d{3,4}[-./_:() ]*\d{4}(?!\d)` |
| 전화번호(유선) | `[전화번호]` | `(?<!\d)(?:02\|031\|032\|033\|041\|042\|043\|044\|051\|052\|053\|054\|055\|061\|062\|063\|064\|070\|080\|050\|060)[-./_:() ]*\d{3,4}[-./_:() ]*\d{4}(?!\d)` |
| 카드번호 | `[카드번호]` | `(?<!\d)(?:\d{4}[-./_:() ]*\d{4}[-./_:() ]*\d{4}[-./_:() ]*\d{4}\|\d{4}[-./_:() ]*\d{6}[-./_:() ]*\d{5})(?!\d)` |
| 여권번호 | `[여권번호]` | `(?<![A-Za-z0-9])[A-Za-z]\d{8}(?![A-Za-z0-9])` |
| 운전면허번호 | `[운전면허번호]` | `(?<!\d)(?:1[1-9]\|2[0-6])[-./_:() ]*\d{2}[-./_:() ]*\d{6}[-./_:() ]*\d{2}(?!\d)` |
| 전자메일 | `[전자메일]` | `[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}` |

> ⚠️ 복사는 **렌더링된 페이지에서** 하세요 — 그러면 `|` 가 올바르게 복사됩니다.
> md 원문 파일에서 직접 복사하면 표 문법 때문에 `\|` 로 이스케이프돼 있어 그대로 쓰면 안 됩니다.

## 정규식 필터로 넣지 않는 것

아래 항목들은 단순 정규식으로 옮기면 오탐이 폭발하거나 형태만으로 판별이 불가해 **프리셋에서 제외**했습니다.

- **부동산 정보 / 인사 정보** 같은 의미 범주: 워크플로우의 **LLM 의미분류** 가 담당 (가이드 3절)
- **계좌번호 / 사업자번호 / 우편번호**: 주변 키워드 문맥이 있어야 판별 가능 (조건부 로직 필요)
- **상세주소 / 금액 / 이름(고객명)** 등: 형태로 판별 불가 (NER 계열 필요)
- 필요 시 운영이 인스턴스에 필터를 추가하면 워크플로우 수정 없이 자동 반영됩니다(가이드 3절 참고).

## 한계 / 주의

- GenOS 정규식 필터는 순수 정규식이라 코드 검증을 못 합니다:
  - 카드번호 **Luhn 체크 없음** → 16자리/4-4-4-4 형태면 무조건 매칭(오탐 가능, 안전 방향)
  - 주민번호는 월/일 범위까지 정규식에 반영해 오탐을 줄였으나 "실존 번호" 검증은 불가
- 즉 이 프리셋은 **과탐(더 가림) 쪽으로 치우침** — 마스킹 목적엔 안전한 방향입니다.

## 검증 이력

- 7종 전부 Python `re` 컴파일 + 양성/음성 샘플 테스트 통과
  (붙은형·하이픈·공백·점 구분자 매칭 / 랜덤 13자리·무효 월일·`@` 없는 문자열 등 음성 거부 확인)
- 운영 인스턴스(99) 등록본으로 전처리기 e2e(실 문서 5종, 표 셀 포함) 마스킹 정상 확인 (2026-07)
